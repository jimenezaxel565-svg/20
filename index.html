Composición sobre Herencia

Deja de crear componentes gigantes con 20 props. La propiedad children es tu mejor herramienta de arquitectura.

<Button variant="primary" size="lg" ... />
Props explosion
Anti-patrón común

Demasiadas props

// ❌ Componente rígido y sobrecargado
function Button({ variant, size, disabled, fullWidth, icon, children }) {
  let className = `btn btn-${variant} btn-${size}`;
  if (fullWidth) className += ' w-full';
  return <button className={className} disabled={disabled}>
    {icon && <span>{icon}</span>}
    {children}
  </button>
}

Problemas: cada nueva variante requiere más props, lógica condicional y pruebas. El componente crece sin control.

{children} es un agujero de composición
Solución básica

children para contenido

// ✅ Un componente simple que solo define estructura
function Card({ children }) {
  return <div className="card">{children}</div>;
}

// Uso: cualquier contenido interno
<Card>
  <h2>Título</h2>
  <p>Contenido libre</p>
  <button>Click</button>
</Card>

children permite inyectar cualquier JSX. El componente padre no necesita saber qué hay dentro.

Header, Body, Footer
Composición avanzada

Varios puntos de inserción

// Componentes pequeños y específicos
function Card({ children }) {
  return <div className="card">{children}</div>;
}
function CardHeader({ children }) {
  return <div className="card-header">{children}</div>;
}
function CardBody({ children }) {
  return <div className="card-body">{children}</div>;
}

// Uso: legible, sin props de configuración
<Card>
  <CardHeader><h2>Mi título</h2></CardHeader>
  <CardBody><p>Contenido real</p></CardBody>
</Card>

Cada subcomponente se encarga de su propio estilo y lógica. No hay props booleanas para mostrar/ocultar partes.

Código más mantenible
Ventajas

¿Qué ganas con composición?

Menos props → componentes más predecibles

Más reutilización → mezcla y combina piezas

Tipado más simple → cada pieza tiene sus propias props

Fácil de testear → componentes aislados

Sin herencia → evitas jerarquías rígidas

Regla de oro
Conclusión

Composición consciente

Regla simple:
¿Tu componente recibe más de 5 props?
→ Piensa en composición con children.
¿Tienes props booleanas para mostrar partes?
→ Mejor crea subcomponentes.

children no es magia, es arquitectura. El código declarativo y compuesto escala mejor que la herencia.
